Phase 2 — Frontend
Day 2 · Plain HTML + JS (no frameworks)
3 pages: index, add, status · Fetch calls from script.js · Status page shows version + load button · No CSS frameworks needed, keep it clean and simple
Phase 3 — Docker
Day 3 · Dockerfile + Docker Compose
Write Dockerfile (python:3.11-slim, gunicorn, port 8000) · docker-compose.yml with app + MongoDB services · Test: docker compose up, verify app runs · Tag image with BUILD_NUMBER env var
Phase 4 — Kubernetes
Days 4–5 · minikube + manifests
minikube start --cpus=4 --memory=6144 · Enable metrics-server addon (critical for HPA) · Write deployment.yaml: 2 replicas, CPU requests set, liveness + readiness probes pointing to /health and /ready · Write service.yaml: NodePort · Write hpa.yaml: min 1, max 5, cpu 50% · kubectl apply -f k8s/ and verify pods running
Phase 5 — Jenkins
Days 6–7 · Pipeline + webhook
Run Jenkins in Docker (mount docker.sock) · Add credentials: Docker Hub (user+pass), kubeconfig (secret file) · Write Jenkinsfile: Clone → Test → Build → Push → Deploy (6 stages) · Set up GitHub webhook via ngrok · Verify end-to-end: push commit → watch pipeline run → new pod appears
Phase 6 — Demo + Docs
Days 8–9 · Polish + report
Rehearse full demo flow · Architecture diagram · README with setup steps · Record demo video as backup · Write report sections

Demo script (what to show in sequence):
1. Open /status — show version v1, build number, pod hostname
2. Push a code change (bump version to v2) → Jenkins pipeline triggers automatically
3. Watch pipeline stages pass in Jenkins UI
4. Refresh /status → version now shows v2 (proves deployment worked)
5. Click "Trigger Load" → watch kubectl get pods -w scale from 1 to 3–5 pods
6. Stop load → pods scale back down (takes ~3–5 min)
7. kubectl delete pod [name] → show it restart automatically
